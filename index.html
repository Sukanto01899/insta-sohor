How Work javascript:- Javascript is an interpreted, single-threaded programming language. The main function is called from the call stack when javascript is executed first. Then all the code inside the file is run one by one. The function contained in the file is added to the call stack one after the other. The call stack is a data structure that records where we are. The call stack is the first in last out system data structure. That means the function that calls first will be executed last.

Whats is event loop:- The event loop is the secret behind JavaScriptâ€™s asynchronous programming. The asynchronous code does not run on the call stack. Asynchronous code transfer to web APIs. Web API processes the code and transfers it to the callback queue. Event loop usually between the Call stack and callback queue. The event loop always checks if the call stack is empty. If the event loop detects that the call stack is empty, the code from the callback queue is moved to the call stack and executed.

Differant between localstorage and session storage;- If we set data on local storage, it will be not removed when we close the browser, it will be saved for a long time. but when we set data on session storage, it will be removed on when we close the browser.
